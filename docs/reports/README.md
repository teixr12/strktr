# STRKTR Reports Directory

This directory stores operational reports generated by local scripts and release workflows.

## Report groups

- `production-audit-*.md`
  - Output of `npm run audit:production`.
  - Includes health, release marker, and Supabase safety audits (when credentials are available).

- `post-wave2-closure-*.md`
  - Output of `npm run ops:post-wave2` or `npm run ops:post-wave2:full`.
  - Includes Wave 2 closure checklist, monitoring window, and rollback drill evidence.

## Usage guidance

- Keep reports versioned during beta for traceability.
- Prefer generating a new report per release window.
- Do not manually edit historical reports; generate a new one instead.
